# Uber Trips Data Analysis

## Project Overview
This project focuses on analyzing Uber trip data to understand user behavior, trip patterns, and key differences between business and personal rides.

The goal was to identify trends and generate insights that could support data-driven decision-making.

---

## Dataset
The dataset includes information about Uber trips such as:
- Start and end time  
- Trip duration  
- Distance (miles)  
- Category (Business / Personal)  
- Purpose of the trip  

---

## Tools Used
- Google Sheets  
- Excel (formulas, pivot tables)  

---

## Analysis Performed

### 1. Data Cleaning & Preparation
- Checked for missing and invalid values  
- Calculated trip duration  
- Created additional columns (time categories, trip purpose)

---

### 2. Trip Analysis by Category
- Compared Business vs Personal trips  
- Calculated:
  - Number of trips  
  - Average duration  
  - Average distance  

---

### 3. Purpose Analysis
- Analyzed trips by purpose (Meeting, Customer Visit, etc.)  
- Identified most frequent trip types  

---

### 4. Time-Based Analysis
- Trips by day of week  
- Monthly trends  
- Time-of-day patterns  

---

## Key Insights

- Business trips account for the majority of rides (over 70%)  
- Most common purposes:
  - Meetings  
  - Customer visits  
  - Errands  

- Business trips show peak activity on weekdays (especially Monday and Friday)  
- Personal trips are more evenly distributed  

- Morning hours (8:00–11:00) show the highest activity  
- Evening trips tend to be longer  

---

## Recommendations

- Improve classification of trips (reduce "NOT DEFINED" category)  
- Use time-based patterns for better resource planning  
- Optimize scheduling based on peak business hours  

---

## Conclusion
This project demonstrates how data analysis can uncover patterns in user behavior and provide actionable insights for business decisions.
